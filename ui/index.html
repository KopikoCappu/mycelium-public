What Mycelium does
One command gives AI agents queryable memory of your entire codebase. Agents call /preflight before touching any file — instead of reading 40 files, they read 4.
CLI commands
mycelium init — scan + AI-describe all files, build graph
mycelium scan — re-scan structure (fast, skips AI descriptions)
mycelium scan --force — full rescan + regenerate all descriptions
mycelium start — start MCP HTTP server and open this viewer
mycelium start --port 4000 — start on a custom port
mycelium status — print graph stats and version
Setting a task
Type your goal in the topbar input (e.g. "add Stripe checkout") and press → or Enter. Mycelium scores every file and highlights the most relevant ones. Your AI agent can call /preflight?task= to get the same list programmatically — so the graph and the agent always agree on what to touch.

Flow dots (when a node is selected)
● Lime — if A imports B, dot travels B→A (dependency source flows into you)
● Pink — if C imports A, dot travels A→C (you flow out to your dependents)
● Red edges — circular import (A imports B imports A)
Node size — bigger = more lines of code and more connections (hot files)
Node color — which top-level directory the file lives in
History tab
Shows every file save during active Mycelium sessions, grouped by task. Summary stats at the top. Click any entry to jump to that file in the graph. Use ↻ Refresh to pull the latest.
MCP / agent endpoints
/preflight?task= — relevant files for a natural-language task
/search?q= — semantic keyword search across all descriptions
/dependencies?file= — what a file imports and who imports it
/xref?file=&fn= — callers and callees of a specific function
/graph — full graph JSON (nodes + edges)
/history — file save log by task and agent
/status — server version and graph stats
Keyboard shortcuts
/ — focus search bar
Esc — close modal → clear selection → clear search (in priority order)
